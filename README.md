# RAG Assistant with Guardrails

> Ask questions over a security / AI-governance corpus (OWASP LLM Top 10 +
> NIST AI RMF). Retrieval-augmented, **evaluated**, and **hardened against the
> OWASP LLM Top 10**.

## Why this exists

Most RAG demos hallucinate and fold to a one-line prompt injection. This one is
**measured** and **defended**: an eval harness reports a scorecard you can
re-run with one command, and a guardrail layer blocks a documented 50-prompt
injection/jailbreak suite — with the false-positive rate tracked so it doesn't
block real questions.

## Architecture

![architecture](docs/architecture.png)

See [docs/architecture.md](docs/architecture.md). Two separate flows: an offline
ingestion pipeline and an online query pipeline that serves answers through the
guardrail layer.

```
User → /chat → [input guardrails] → retriever (pgvector) → prompt (instruction
isolation) → LLM (Claude) → [output guardrails: PII redaction, grounding gate] →
{ answer, citations, latency, guardrail_flags }
```

## Run it in 2 minutes (Docker)

```bash
git clone https://github.com/ousmaneb/rag-guardrails && cd rag-guardrails
cp .env.example .env        # add ANTHROPIC_API_KEY (optional — see note below)
docker compose up --build   # API on :8000, UI on :8501, pgvector on :5432
```

Then open the UI at <http://localhost:8501> or the API docs at
<http://localhost:8000/docs>.

> **No API key?** The app still runs end-to-end: with no `ANTHROPIC_API_KEY` the
> LLM falls back to a deterministic "echo" provider and embeddings fall back to a
> hashing embedder. Great for trying the plumbing and for CI — add a key for
> real answers and real eval numbers.

## Run it locally (no Docker)

```bash
python -m venv .venv && source .venv/bin/activate
pip install ".[dev,eval,ui]"
cp .env.example .env

# Use the in-memory vector store + offline embeddings for a zero-dependency run:
export VECTOR_STORE=memory EMBEDDINGS_PROVIDER=hash

python -m rag.ingest                 # build the index
uvicorn rag.api:app --reload         # API on :8000
streamlit run ui/streamlit_app.py    # UI on :8501  (separate terminal)
python eval/run_eval.py --reingest   # reproduce the scorecard
```

`make dev install ingest api ui eval test lint up down` are provided as shortcuts.

## Evaluation harness

```bash
python eval/run_eval.py --label baseline   # then change config in .env...
python eval/run_eval.py --label best       # ...and compare the delta
```

- **Quality** (`eval/questions.yaml`): faithfulness, answer relevancy, context
  recall/precision, p95 latency. Uses **RAGAS** when a real LLM is configured;
  otherwise a self-contained lexical scorer so it always runs.
- **Security** (`eval/attacks.yaml`): a 50-prompt suite → block rate + false
  positives, broken down by attack category. Survivors are listed honestly.
- Runs are logged to **MLflow** when it's installed (`mlruns/`).

Tune `chunk_tokens`, `top_k`, the model, and the guardrail thresholds in `.env`,
re-run, and keep the config that wins.
